# Modular Weekly Study Templates

To manage a **21-month preparation timeline** without accumulating operational fatigue, your weekly execution cannot be static. Depending on corporate workload shifts and exam proximity, deploy one of these four standardized modular templates.

---

## 🧭 Weekly Macro Architecture Overview

```mermaid
graph TD
    subgraph Modular Weekly Execution Profiles
        Standard[Template A: Standard Production Week<br>Target: Steady Baseline Ingestion]
        Heavy[Template B: Heavy Implementation Week<br>Target: Algorithmic Tracing / Coding]
        Revision[Template C: High-Frequency Revision Week<br>Target: Defect Sweeps / Active Recall]
        Recovery[Template D: Regulated Debt Triage Week<br>Target: Buffer Isolation / Backlog Catchup]
    end

    Standard -->|Corporate Project Crunch| Revision
    Heavy -->|Weekend Mock Scheduled| Standard
    Recovery -->|Backlog Cleared| Standard
```

---

## 📋 Template A: Standard Production Week
*Optimal state: Steady workday shift; balanced energy baseline.*

- **Monday - Friday Desk Blocks (5 Hours Total):** Primary text chapter ingestion + real-time short-note extraction sweeps.
- **Monday - Friday Commute Blocks (10 Hours Total):** Active digital flashcard verification + offline formula parsing.
- **Saturday Execution (8 Hours):** Deep morning synthesis + immediate chapter PYQ extractions.
- **Sunday Execution (8 Hours):** Integrated 90-minute Sectional Test + complete analytical post-test reviews.

---

## 📋 Template B: Heavy Implementation Week
*Optimal state: Scheduled for Core DSA, Python/C++ loops, or sliding window protocols.*

- **Weekday Desk Blocks:** Direct paper coding derivations. Trace out state transitions manually before digital verification.
- **Weekday Commute Blocks:** Scan visual trace trees and binary matrix memory layouts via tablet arrays.
- **Saturday Execution:** Complete local terminal execution of multi-step back-tracking algorithms.
- **Sunday Execution:** Topic testing + double precision numerical answer parsing sweeps.

---

## 📋 Template C: High-Frequency Revision Week
*Optimal state: Final phase consolidation windows leading up to exams.*

- **Weekday Desk Blocks:** Zero fresh chapter reading. Execute double-pass timed sectional tests exclusively.
- **Weekday Commute Blocks:** High-speed parsing of **Layer 2 Ultra-Short Revision Sheets** + inverse formula checks.
- **Weekend Execution:** Consecutive Full-Length Mock testing arrays alongside exhaustive mistake logging cycles.

---

## 📋 Template D: Regulated Debt Triage Week
*Optimal state: Following multi-day corporate travel overruns or physical recovery loops.*

- **Weekday Operations:** Strip active production modules completely. Transition commute times strictly to passive mental validation of existing notes.
- **Weekend Execution:** Deploy **Debt Compression Mechanics** ([08_backlog_recovery.md](./08_backlog_recovery.md)) to extract the minimum viable scoring core from quarantined backlog files.
